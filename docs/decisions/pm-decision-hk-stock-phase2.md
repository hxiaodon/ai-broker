# PM 决策：港股交易推迟至 Phase 2

> **决策编号**: DEC-2026-003
> **决策日期**: 2026-03-12
> **决策人**: PM
> **状态**: APPROVED
> **影响范围**: 设计文档 v3-final、HTML 原型、交易引擎、出入金、行情服务

---

## 决策

**Phase 1 (MVP) 仅支持美股交易，港股交易整体推迟至 Phase 2。**

---

## 背景

技术部统一评审意见（`docs/review/tech-department-consolidated-opinion.md`）指出港股适配增加约 30% 工作量。UX 设计师评审（`docs/review/design-review-for-pm.md`）指出港股细节几乎为零。PM 原型评审（`docs/review/pm-prototype-review.md`）确认港股相关设计不足以支撑开发启动。

## 决策依据

| # | 理由 | 详情 |
|---|------|------|
| 1 | **交易规则根本性差异** | Board Lot（每手股数按股票不同：100/200/400/500/1000/2000）、Tick Size 按价格区间浮动、竞价时段（09:00-09:30 开市前、16:00-16:10 收市竞价）— 均未设计 |
| 2 | **HKD 出入金通道空白** | FPS（转数快）是港股零售市场基础设施，完全未设计；HKD Wire 入金流程也未涉及 |
| 3 | **T+2 结算差异** | 美股已切换至 T+1（2024年5月），港股仍为 T+2，出金可用余额计算逻辑不同 |
| 4 | **交易引擎工期** | 交易引擎评估港股适配增加 ~8-10 周；FIX Protocol 对接 HKEX 需单独 UAT 认证 |
| 5 | **聚焦核心闭环** | 先用美股跑通"注册→KYC→入金→交易→出金"完整闭环，降低 Phase 1 风险 |

## Phase 1 范围变更

### 移除项

| 模块 | 移除内容 | 处理方式 |
|------|---------|---------|
| 行情页 | 港股 Tab 内容 | Tab 保留但显示"即将上线"占位页 |
| 出入金 | HKD 币种切换、换汇功能 | 移除 HKD 按钮；换汇入口改为"即将上线" |
| 交易页 | 港股下单（Board Lot/Tick Size/竞价时段） | 不开发 |
| KYC | 港股特有合规项（SFC 额外要求） | 推迟，Phase 1 仅满足 SEC/FINRA 要求 |
| 持仓 | HKD 持仓展示 | 仅显示 USD 资产 |

### 保留项（为 Phase 2 预留扩展性）

| 模块 | 保留内容 | 原因 |
|------|---------|------|
| 行情页 Tab | "港股" Tab 入口 | 用户可见产品规划，点击显示"即将上线" |
| 数据库 Schema | 多币种字段设计 | 后端架构设计时预留，避免 Phase 2 大规模迁移 |
| KYC | 香港身份证（HKID）证件类型 | 已在 Step 1 设计中，保留不增加成本 |

## 工期影响

| 角色 | Phase 1 原估 | 去除港股后 | 节省 |
|------|-------------|-----------|------|
| 交易引擎 | 5 个月 | 3.5-4 个月 | ~4-6 周 |
| 出入金 | 5 个月 | 3.5-4 个月 | ~4-6 周 |
| iOS/Android | 各 5-6 个月 | 各 4-5 个月 | ~4 周 |
| 后端 | 4-5 个月 | 3.5-4 个月 | ~3-4 周 |

## Phase 2 港股计划（Month 5-8）

Phase 2 需补充完整的港股设计，包含但不限于：
- Board Lot 规则查询接口和下单校验
- 港股竞价时段下单 UI 和订单类型（竞价盘/竞价限价盘）
- FPS 入金通道设计和实现
- HKD/USD 换汇功能
- T+2 结算在出金可用余额的体现
- HKEX 行情数据授权和接入
- 港股相关合规（SFC 投资者知识评估、VCM 波幅调节机制提示）
