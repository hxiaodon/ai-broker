## 一、6 项 CRITICAL 问题回复

### CRITICAL-1: PDT 开关必须删除 ✅ 同意

**决策**: 删除 v3 交易设置页中的"显示 PDT 警告"开关。PDT 是 FINRA 强制要求，实现为**硬拦截**（交易引擎层面拒绝订单），前端不可绕过。

**保留**: PDT 规则说明页面（从交易设置中可访问），仅作为教育用途。

### CRITICAL-2: Margin Call 设计缺失 ✅ 同意

**决策**: **MVP 仅支持现金账户（Cash Account）**。从 KYC 流程中移除"账户类型选择"步骤。Margin 账户（含 Margin Call、维持保证金监控、强制平仓）推至 Phase 2（Month 5-8）。

**理由**: Margin 系统复杂度极高（Reg T 计算、DTBP、利息系统），在 6 个月期限内加入会严重影响交付质量。

### CRITICAL-3: 入金 AML 筛查缺失 ✅ 同意

**决策**: 入金流程增加"合规审核中"状态节点。

**UX 处理**:
- 正常情况（>99%）：后台秒级通过，用户无感知，直接进入"银行处理中"
- 筛查结果为 REVIEW：展示"资金合规审核中，预计 1-2 个工作日"状态
- 筛查结果为 BLOCKED：展示"入金申请未通过，请联系客服"，推送通知

### CRITICAL-4: Travel Rule 信息收集 ✅ 同意

**决策**:
- Wire 入金：后端从银行 SWIFT 消息自动提取 originator/beneficiary 信息，无需用户额外填写。后端校验完整性，不完整则标记人工审核。
- Wire 出金：使用用户 KYC 已有信息作为 originator 信息。KYC 信息不完整时提示用户先完善个人资料。

### CRITICAL-5: Admin Panel 同步立项 ✅ 同意

**决策**: Admin Panel MVP 与移动端同步开发，纳入 Phase 1。

**Phase 1 Admin Panel 范围**:
| 模块 | 功能 |
|------|------|
| KYC 审核工作台 | 审核队列、证件查看、审批/拒绝/要求补材料 |
| 出金审批 | 审批队列、三级审批流程、审批记录 |
| 用户管理 | 搜索、查看详情、冻结/解冻账户 |
| 订单监控 | 实时订单大屏、异常订单告警 |
| 系统仪表盘 | 核心指标看板（注册数、入金额、交易量） |

**Phase 2 补充**: 合规报告中心（CTR/SAR）、对账系统、营销管理、完整 CRM。

### CRITICAL-6: KYC SDK 选型立即启动 ✅ 同意

**决策**: 本周启动评估。**但 Phase 1 不强制要求人脸识别 SDK**。

**Phase 1 方案**: 证件上传 + OCR 识别 + **人工审核**（替代活体检测），降低 SDK 依赖。
**Phase 2 方案**: 接入活体检测 SDK（Onfido / Sumsub），实现自动化 KYC。

**本周行动**: 同步启动 SDK 商务评估（Onfido、Sumsub、Jumio），为 Phase 2 提前锁定供应商。但 Phase 1 开发不被 SDK 合同阻塞。

## 二、20 项待决策问题逐一回复

### 2.1 MVP 范围决策

| # | 问题 | 决策 | 理由 |
|---|------|------|------|
| 1 | 现金 vs 保证金账户 | **现金账户 Only** | 见 CRITICAL-2。保证金 Phase 2 |
| 2 | KYC 9 步全做？ | **简化为 7 步** | 见下方详细说明 |
| 3 | 港股 MVP 首发？ | **是，但简化规则** | 见下方详细说明 |
| 4 | 即时入金（垫资）MVP？ | **否，Phase 3** | 信用风险模型、Clawback 复杂度过高 |
| 5 | 订单修改 MVP？ | **否，仅支持撤单** | 改单=撤单+重下，竞态复杂。从 UI 移除"修改"按钮 |

**KYC 简化为 7 步**:

| 步骤 | 内容 | 变更 |
|------|------|------|
| 1. 个人信息 | 姓名、出生日期、国籍、证件号 + 就业状态（简化合并） | 就业详情从独立步骤合并到此步骤，仅保留就业状态 + PEP 声明 |
| 2. 证件上传 | 拍照上传 + OCR 识别 | 不变 |
| 3. 财务状况 | 年收入、净资产、资金来源 | 不变 |
| 4. 投资评估 | 投资经验、风险承受能力 | 不变 |
| 5. 税务声明 | W-8BEN / W-9 签署 | 不变 |
| 6. 风险披露 | 5 份风险文件确认 | 不变 |
| 7. 协议签署 | 客户协议 + 隐私政策 + 数据协议 + 电子签名 | **电子签名改为复选框 + 姓名输入确认**（去掉手写签名） |

**去掉的步骤**: 人脸识别（Phase 1 用人工审核替代）、独立的就业信息步骤（合并）、手写签名（简化为 typed name）。

**港股 MVP 简化方案**:

| 项目 | Phase 1 范围 | Phase 2 补充 |
|------|-------------|-------------|
| 交易时段 | 常规交易时段 only (09:30-12:00, 13:00-16:00 HKT) | 开市前竞价 + 收市竞价 |
| 订单类型 | 限价单 + 市价单 | 增强竞价单 |
| 入金通道 | Wire (CHATS) | FPS 实时入金 |
| 碎股交易 | 不支持（整手交易） | P1 支持碎股 |
| Board Lot | 从标的信息服务动态获取，下单页验证 | 不变 |
| Tick Size | 按 HKEX 价格区间表验证 | 不变 |

## 四、MVP Phase 分阶段确认

综合以上决策，确认技术部建议的 3 Phase 方案，**微调如下**：

### Phase 1: 双市场核心闭环（Month 1-5）

> 目标: 注册 → KYC → 入金 → 买卖美股/港股 → 出金

| 模块 | 范围 |
|------|------|
| 登录注册 | 手机号+OTP，+86/+852，生物识别快捷登录 |
| KYC | 7 步简化版（人工审核，无活体检测），Tier 1/2 |
| 行情 | 美股+港股实时行情（注册用户），延迟行情（游客），L1 only |
| 交易 | 市价单+限价单，DAY/GTC，美股全时段+港股常规时段 |
| 持仓 | 持仓列表、实时 P&L、资产总览（双币种） |
| 订单 | 订单列表、状态追踪、撤单（无改单） |
| 出入金 | USD: Wire入+ACH出；HKD: Wire(CHATS)入出。银行卡管理 |
| Admin Panel | KYC审核+出金审批+用户管理+订单监控+仪表盘 |

### Phase 2: 功能完善（Month 5-8）

| 模块 | 新增 |
|------|------|
| KYC 完整版 | 活体检测 SDK + OCR 自动审核 + 手写签名 |
| 保证金账户 | Margin Call + 维持保证金 + 强制平仓 |
| 港股增强 | 竞价时段 + 碎股交易 + FPS 入金 |
| 订单增强 | 止损/止损限价/追踪止损 + 改单 + IOC/FOK |
| 换汇 | USD⇄HKD 换汇页面 |
| 邮箱登录 | 邮箱+密码备选登录方式 |
| Admin 增强 | 合规报告(CTR/SAR) + 对账系统 |

### Phase 3: 高级功能（Month 8-10）

| 模块 | 新增 |
|------|------|
| 即时入金（垫资） | 信用评估 + Clawback |
| L2 深度行情 | 5/10 档盘口 |
| 技术指标 | MACD / RSI / BOLL |
| 通知中心 | App 内消息收件箱 |
| 平台增值 | iOS Widget / Live Activities |
