## 一、已完成的修改（PM 已直接修改原型代码）

以下修改已在原型 HTML 文件中完成，UXUE 需要在 Figma/设计稿中同步更新：

### 1. 港股推迟至 Phase 2

**决策**: Phase 1 (MVP) 仅支持美股交易，港股整体推迟至 Phase 2。

**已修改的原型文件**:

| 文件 | 修改内容 |
|------|---------|
| `market.html` | 港股 Tab 点击后显示"港股交易即将上线"占位页（含 HKEX 图标和说明文字） |
| `funding.html` | HKD 余额区域改为"即将上线"灰化状态；换汇卡片改为"即将上线"灰化状态；入金币种 HKD 按钮 disabled 并标注"即将上线" |

**设计稿需同步**:
- 行情页港股 Tab 的"即将上线"占位页设计
- 出入金页面 HKD 相关区域的灰化/占位设计
- 搜索结果页仅返回美股（移除港股示例数据）
- 持仓页移除 HKD 相关展示

### 2. 出金流程添加 Face ID 验证

**已修改文件**: `funding.html`

**修改内容**:
- 出金"确认出金"按钮点击后不再直接提交，而是弹出确认弹窗（Bottom Sheet）
- 弹窗包含：出金摘要（金额、方式、目标银行、手续费、实际到账、预计到账）
- 同名账户警告提示
- "确认并提交 (Face ID)" 按钮（含 Face ID 图标）
- "取消"按钮
- 与入金确认流程保持一致的安全验证体验

**设计稿需同步**:
- 出金确认 Bottom Sheet 的完整视觉设计
- Face ID 验证中/成功/失败状态设计
- 确保与入金确认弹窗保持视觉一致性

### 3. 游客模式延迟行情标识

**已修改文件**: `market.html`、`stock-detail.html`、`login.html`

**修改内容**:
- `login.html`: "先浏览行情"链接现在带 `?guest=1` 参数
- `market.html`: 游客模式下顶部显示琥珀色通知栏"行情延迟 15 分钟，登录获取实时行情"+ 去登录入口；每个股票价格旁显示"延迟"徽标
- `stock-detail.html`: 游客模式下顶部显示同样的延迟通知栏；价格旁显示"延迟"徽标；底部买入/卖出按钮替换为"登录后即可交易"+ 登录/注册按钮

**设计稿需同步**:
- 延迟通知栏的完整视觉设计（琥珀色背景 + "延迟"标签 + 文案 + 登录入口）
- "延迟"价格徽标的样式规范（尺寸、颜色、位置）
- 游客模式下的底部交易按钮替换设计（登录引导 Bottom Sheet）
- 确保延迟标识在所有价格展示位置统一

### 4. stock-detail.html 买入/卖出按钮

**确认**: 经核查，原型中 stock-detail.html **已有** 底部固定的买入/卖出按钮（绿色买入 + 红色卖出），链接到 trade.html。此项无需修改。

## 三、设计规范补充

在适配过程中，请遵循以下新增的设计规范：

### 延迟标识规范

```
标签样式:
- 背景: amber-200 (#FDE68A)
- 文字: amber-900 (#78350F)
- 字号: 9px / 10px
- 圆角: 4px
- 内距: 4px 6px
- 文案: "延迟"

通知栏样式:
- 背景: amber-50 (#FFFBEB)
- 边框底部: amber-200
- 内距: 8px 16px
- 高度: 约 36px
- 文案: "行情延迟 15 分钟，登录获取实时行情"
- 右侧: "去登录" 文字链接 (blue-600)
```

### "即将上线" 占位页规范

```
图标: 蓝色圆形背景 (blue-50) + 时钟图标 (blue-400)
标题: "XX功能即将上线" (16px semibold, gray-700)
说明: 简要说明 (14px regular, gray-500)
垂直居中显示
```

### 出金确认弹窗规范

```
类型: Bottom Sheet
圆角: 顶部 16px
背景遮罩: black/50
内容:
- 拖动手柄 (居中)
- 标题: "确认出金信息" (18px bold)
- 摘要卡片: gray-50 背景, 12px 圆角
- 同名账户警告: yellow-50 背景, yellow-200 边框
- 主按钮: "确认并提交 (Face ID)" (blue-600, 含 Face ID 图标)
- 取消按钮: 纯文字 (gray-500)
```
