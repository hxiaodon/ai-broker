# 产品方案 Review 报告 — 上线缺失功能分析

> 从产品经理视角对初版方案的全面审查，聚焦"真实上线还缺什么"
>
> 审查范围: mobile-app-design-v2.md / trading-system.md / fund-transfer-system.md / market-data-system.md

---

## 总体评价

现有方案在**交易核心引擎、行情系统、出入金系统**三大技术架构上已经比较完整，移动端 UI 设计覆盖了主要交易流程。但作为一个真正要上线的美港股券商 App，仍有**多个监管强制要求和关键产品能力缺失**。

---

## 一、P0 — 监管强制 / 上线必须（缺一不可）

### 1. W-8BEN / W-9 表格管理
- **现状**：仅在股息扣税代码中提到 `TaxWithholdingRate`，但无表格提交流程
- **问题**：非美国税务居民买美股前**必须提交 W-8BEN**，否则股息预扣税 30%（有税收协定可降至 10%）；美国居民需 W-9
- **建议**：KYC 流程中增加税务身份声明步骤，支持电子签署 W-8BEN，W-8BEN 三年有效需定时提醒续签
- **状态**: ✅ 已在 v3 补充设计中补齐 (KYC Step 7)

### 2. 行情数据协议签署
- **现状**：行情系统架构完整，但未涉及用户端的数据授权
- **问题**：NYSE/NASDAQ 要求用户在获取实时行情前必须签署 **Market Data Subscriber Agreement**，声明是否为 Professional 用户。HKEX 同理
- **建议**：首次开启实时行情时弹出协议签署页；区分 Professional vs Non-Professional 定价；未签署用户只展示 15 分钟延迟行情
- **状态**: ✅ 已在 v3 补充设计中纳入 (KYC Step 9 协议签署)

### 3. 交易确认书（Trade Confirmation）
- **现状**：通知系统仅提及 Push/SMS/Email 通知，无交易确认书
- **问题**：SEC Rule 10b-10 要求每笔成交后必须向客户发送**交易确认书**，包含成交价格、数量、佣金明细、Best Execution 信息等
- **建议**：App 内增加"交易确认"模块，每笔成交自动生成 PDF 确认书，支持 App 内查看和邮件发送
- **状态**: ⏳ 需要独立设计（报表模块）

### 4. 月结单 / 季度报告 / 年度税务报表
- **现状**：完全缺失
- **问题**：FINRA Rule 2231 要求至少每季度发送账户报表；年底需生成 1042-S（非美国居民）或 1099（美国居民）税表
- **建议**：
  - 月结单：包含期间交易汇总、持仓变动、费用明细、期末余额
  - 年度税务报表：1042-S / 1099-DIV / 1099-B
  - 支持 App 内查看 + PDF 下载 + 邮件推送
- **状态**: ⏳ 需要独立设计（报表模块）

### 5. 账户注销/关户流程
- **现状**：完全缺失，只有账户创建
- **问题**：监管要求用户有权关闭账户；涉及持仓清算、资金全部出金、数据保留等复杂流程
- **建议**：设计关户申请 → 清仓检查 → 全部出金 → 数据归档（保留 7 年）→ 账户停用流程
- **状态**: ⏳ 需要独立设计

### 6. KYC 年度复核
- **现状**：仅有首次 KYC 流程
- **问题**：AMLO / SFC 要求对客户进行持续尽职调查（Ongoing CDD），高风险客户至少年度复核
- **建议**：定时触发 KYC 刷新提醒，过期未更新的账户限制交易功能
- **状态**: ⏳ 需要后台系统设计

### 7. SAR/CTR 报告工作流
- **现状**：出入金合规规则中提及 CTR/SAR，但无实际工作流设计
- **问题**：CTR >$10,000 必须在 15 天内向 FinCEN 提交；SAR 需要合规官审批流程
- **建议**：设计自动 CTR 生成 → 合规审核 → FinCEN BSA E-Filing 提交的完整链路
- **状态**: ⏳ 需要后台系统设计

### 8. 客诉处理流程
- **现状**：仅有"帮助中心"入口
- **问题**：FINRA Rule 4513 要求保留所有客户投诉记录 4 年；SEC 要求有书面投诉处理流程
- **建议**：设计投诉提交 → 受理确认 → 调查处理 → 反馈回复的闭环流程，后台需要工单系统
- **状态**: ⏳ 需要独立设计（客服模块）

### 9. 多语言支持
- **现状**：完全缺失
- **问题**：跨境美港股 App，用户必然涉及英文、简体中文、繁体中文。SFC 要求向香港投资者提供中文服务
- **建议**：至少支持 English / 简体中文 / 繁体中文，KYC 协议需提供双语版本
- **状态**: ⏳ 需要全局国际化方案

## 二、P1 — 首发版本强烈建议包含

### 10. IPO / 打新申购
- **现状**：完全缺失
- **问题**：港股打新是散户核心需求之一，也是获客杀手功能。美股 IPO 参与（如通过承销商分配）也是竞争力
- **建议**：设计打新申购 → 冻结资金 → 中签确认 → 分配股份的完整流程

### 11. 公司行动完整覆盖
- **现状**：仅有 Dividend 和 Stock Split
- **问题**：缺少 **合股（Reverse Split）、供股（Rights Issue）、红股（Bonus Shares）、收购要约（Tender Offer）、私有化** 等
- **建议**：设计 Corporate Action 通知 → 用户选择（如供股是否参与）→ 自动执行的完整框架

### 12. 在线客服系统
- **现状**：仅有"帮助中心"
- **问题**：券商客户对资金和交易问题的咨询需要即时响应，纯自助帮助中心不够
- **建议**：
  - FAQ + 智能客服机器人（覆盖 80% 常见问题）
  - 人工客服转接（工单系统 + 在线聊天）
  - 工作时间覆盖美港股交易时段

### 13. 通知中心
- **现状**：仅提及 Push 通知，无 App 内消息收件箱
- **问题**：用户需要在 App 内查看历史通知（成交、出入金、系统公告、风险提醒等）
- **建议**：设计消息中心页面，分类（交易/资金/系统/活动），支持已读/未读、批量处理

### 14. 融资利息 / 现金利息
- **现状**：保证金系统只有保证金率计算，无利息
- **问题**：Margin 账户使用融资买股票需收取**融资利息**（券商重要收入来源）；部分券商会对闲置现金支付利息
- **建议**：
  - 融资利息：日计息、月结算，按 base rate + markup 计算
  - 现金利息：可选功能，增强用户粘性

### 15. 跨币种购买力
- **现状**：账户分 USD/HKD，但未说明跨币种购买力
- **问题**：用户持有 HKD 想买美股怎么办？需要自动换汇还是手动？
- **建议**：
  - 方案 A：自动换汇 + 实时汇率预览
  - 方案 B：手动换汇后再交易
  - 明确换汇手续费和汇率锁定机制
- **状态**: ✅ 换汇页面已在 v3 补充设计中新增

### 16. 股票转入/转出（ACATS / CCASS）
- **现状**：完全缺失
- **问题**：用户从其他券商转户到本平台（或转出），需要支持 ACATS（美股）和 CCASS（港股）转仓
- **建议**：设计转仓申请 → 表格填写 → 后台处理 → 持仓入账的流程（通常 3-7 工作日）

### 17. 网络异常 / 断线重连处理
- **现状**：WebSocket 层面有 reconnect 设计，但客户端 UX 层面未覆盖
- **问题**：交易中网络中断，用户不知道订单是否成功提交，非常危险
- **建议**：断网提示、订单状态确认引导、自动重连 + 状态同步
- **状态**: ✅ 已在 v3 补充设计中补齐（网络异常全局提示）

### 18. 基本面数据
- **现状**：完全缺失
- **问题**：股票详情页仅有行情数据，无公司基本面信息
- **建议**：集成 P/E、市值、EPS、营收、利润、分红历史、行业分类等

### 19. 设备管理
- **现状**：安全规则提及设备绑定，但无用户端管理
- **问题**：用户需要查看登录设备列表、远程登出其他设备
- **建议**："我的" → "安全设置" 中增加已登录设备列表，支持远程登出

### 20. 管理后台 MVP
- **现状**：CLAUDE.md 提及 Admin Panel 但无实际设计
- **问题**：运营团队无后台则无法运行业务
- **建议**：MVP 至少包含：
  - KYC 审核工作台
  - 出金审批队列
  - 用户管理（搜索、查看、冻结）
  - 订单监控大屏
  - 系统监控仪表盘

## 四、总结

| 优先级 | 缺失项数 | 关键类型 | 当前进展 |
|--------|---------|---------|---------|
| **P0（上线必须）** | **9 项** | 监管合规、法律要求 | 3 项已在 v3 补充设计覆盖 |
| **P1（首发建议）** | **11 项** | 核心体验、竞争力 | 2 项已在 v3 补充设计覆盖 |
| **P2（后续迭代）** | **14 项** | 增长、差异化 | 待规划 |

**最关键的 3 个仍未覆盖的 P0 缺失**：
1. **交易确认书 + 月结单 + 税务报表** — SEC 强制要求，需要独立的报表模块设计
2. **SAR/CTR 监管报告工作流** — 合规后台必备，需要完整的工作流设计
3. **多语言支持** — 跨境产品基本要求，影响所有页面
