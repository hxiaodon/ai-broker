```
    │
    ├─ "中央政治局委员"、"副总理"、"部长"、"省长"

    │
    ├─ "副省长"、"市长"、"市委书记"、"央企董事长"
    │  → LEVEL 2（人工评估）
    │     ├─ 工薪+投资 → 无需 EDD（正常激活）
    │     ├─ 资产异常 → 升级 EDD
    │     └─ 频繁大额交易 → 升级 EDD
    │
    ├─ "副市长"、"县长"、"副董事长"
    │  → LEVEL 3（标记，无需 EDD）
    │     └─ 交易监控阈值 -50%（如触发异常 → SAR）
    │
    └─ "镇长"、"科员"、"自营商人"
       → 不是 PEP（正常 KYC）
```

## 3. EDD 评估检查清单（Level 1 必填，Level 2 参考）

### 财富来源合理性评估

```
□ 薪酬信息
  ├─ 最近 3 年的工资条/税单副本
  ├─ 验证：薪酬 > 资产增加幅度？
  └─ 异常标记：年薪 $100K，但流动资产 $5M

□ 投资收益
  ├─ 股票/基金持仓证明
  ├─ 分红/利息收据（最近 3 年）
  └─ 验证：收益率是否合理（10-20% 范围内？）

□ 房产和不动产
  ├─ 房产证/购房合同
  ├─ 贷款证明（如有按揭）
  └─ 估值与市场价格是否匹配？

□ 其他资产来源
  ├─ 家族继承证明（若有）
  ├─ 生意营收证明（若自雇）
  └─ 任何超过 $50K 的资金来源都需文件

□ 红旗信号
  ├─ 资产增加无法解释（年增长 > 50%）
  ├─ 频繁的跨国转账，无明确用途
  ├─ 与官员薪酬明显不符的奢侈消费
  └─ → 如发现 3 个以上红旗，建议拒绝开户或升级 SAR
```

### 高管批准流程

```
合规官员完成评估
    ↓
    ✍️ 签署"EDD 评估报告"（含建议：批准或拒绝）
    ↓
转交合规经理/CEO 进行最终审查
    ↓
高管批准（必须有 CEO 或 COO 级别签署）
    ↓
    ├─ 批准 → 账户激活，保存文件 7 年
    └─ 拒绝 → 发送拒绝通知，说明理由（泛化描述，避免 Tipping-Off）
```

## 5. 合规官员决策矩阵（Level 2 评估用）

当检测到 Level 2 官员时，使用此矩阵快速判断：

```
Level 2 官员识别
    │
    ├─ 财富来源简单吗？
    │  ├─ 仅工薪 + 投资分红 → 继续
    │  └─ 复杂/未知 → 升级 EDD，停止
    │
    ├─ 交易历史是否异常？
    │  ├─ 无交易历史或小额股票交易 → 继续
    │  ├─ 频繁大额进出（>$50K/周）→ 升级 EDD，停止
    │  └─ 多国汇款或加密交易迹象 → 升级 EDD，停止
    │
    ├─ 有无不良新闻报道？
    │  ├─ 无（或仅正面报道） → 继续
    │  ├─ 腐败/贪污指控 → 拒绝开户，停止
    │  └─ 政治风险报道 → 升级 EDD，停止
    │
    └─ 决策结果
       ├─ 通过所有检查 → MEDIUM 风险，不做 EDD，正常激活
       ├─ 任何一项失败 → 升级 EDD 工作流
       └─ 腐败证据明显 → 拒绝开户，记录 SAR
```

## 6. 红旗信号速查表

遇到以下信号时，自动升级风险评估：

| 红旗信号 | 风险等级 | 建议行动 |
|---------|---------|---------|
| **资产无法解释**（年增 > 50%，无来源证明） | 🔴 HIGH | 升级 EDD，可能拒绝 |
| **配偶为官员，但申请人隐瞒** | 🔴 HIGH | SAR（隐瞒行为），拒绝开户 |
| **频繁跨境汇款，无合理用途**（>$100K/周） | 🔴 HIGH | SAR（可疑交易模式） |
| **与制裁国家/地区的商业往来**（伊朗、朝鲜等） | 🔴 HIGH | 立即冻结，SAR |
| **历史上有腐败/贪污指控** | 🔴 HIGH | 拒绝开户 |
| **虚假信息（申报职务与官方档案不符）** | 🔴 HIGH | SAR（诚信问题），拒绝 |
| 资产增加可以部分解释，但有细微差异 | 🟡 MEDIUM | 要求补充说明，可能升级 EDD |
| 交易频繁但额度正常，用途明确 | 🟡 MEDIUM | 持续监控，不升级 EDD |
| 配偶职务为县级或中层管理 | 🟡 MEDIUM | 按主体官员级别处理，可能不升级 |
| 申请人已离职，但仍标记为 PEP | 🟢 LOW | 如离职 >3 年，可考虑降级（但保持监控） |

## 7. 常见客户问题的应答（客服脚本）

### Q1："我是市长，你们为什么拒绝我？"

**标准回答**：
```
感谢您的理解。根据国际反洗钱规定，
我们需要对政府官员进行额外的背景审查（强化尽职调查）。
这不是拒绝，而是需要多花 5-7 个工作日来完成审查。

我们的合规团队正在审视您的财务背景，
完成后会通知您审批结果。
```

### Q2："如果我隐瞒职务会怎样？"

**标准回答**：
```
非常不建议这样做。我们的系统会通过第三方数据库检测您的职务信息。
如果发现隐瞒，我们会：
1. 立即冻结您的账户
2. 向监管部门报告（可疑活动报告）
3. 您可能被列入业界黑名单，影响其他平台开户

诚实披露是最安全的选择。
```

### Q3："为什么市长要做 EDD，但副市长不需要？"

**标准回答**：
```
这是国际监管标准（FATF）的规定。
主要原因是权力范围：

市长（正职）：掌握城市资源配置权、招商权、财政支出权
副市长：通常只管某个具体部门，权力有限

监管部门依据"权力重要性"而非"官职名称"来判断。
```

## 11. 故障排除

### 问题：某官员在 ComplyAdvantage 搜不到

**检查步骤**：
```
1. 确认官员姓名拼音正确（可能有变体，如 Wu/Woo）
2. 等待 ComplyAdvantage 更新（新晋升官员可能 1-2 周延迟）
3. 人工在 World-Check 中搜索确认
4. 若都找不到但申请人声称是官员 → 升级风险评分为 MEDIUM，要求补充证明
```

### 问题：某官员离职了，是否改回 Non-PEP？

**处理步骤**：
```
1. 确认离职时间（官方档案）
2. 如离职 < 3 年：保持 PEP 标记，但监控强度可下调
3. 如离职 > 3 年：可考虑将 pep_level 改为 NULL，但保留 pep_verified_at 历史记录
4. 不建议删除，因为"曾经是 PEP"仍需审计留痕
```

### 问题：某申请人的资产增长无法解释，怎么处理？

**决策树**：
```
资产无法解释（如年增 > 100%）
    │
    ├─ Level 1 官员 → SAR（可疑财务活动），拒绝开户
    │
    ├─ Level 2 官员 → 升级 EDD，要求 3 个月内补充说明
    │  └─ 3 个月后仍无说明 → SAR，拒绝开户
    │
    └─ Level 3 官员 → 标记高风险，加强交易监控
       └─ 如后续交易异常 → SAR
```

## 12. 快速参考：官员身份隐瞒时的 SAR 流程

```
发现申请人隐瞒官员身份
    │
    ▼
立即冻结账户（ACCOUNT_BLOCKED）
    │
    ▼
合规官员编写 SAR（可疑活动报告）
    ├─ 隐瞒身份的事实
    ├─ 涉及资金金额
    ├─ 申请人信息
    └─ 建议原因："Structuring / Identity Concealment"
    │
    ▼
高管审批 SAR
    │
    ▼
向 FinCEN（美股）/ JFIU（港股）提交
    │
    ▼
记录 SAR 报告号，保存 5 年
    │
    ▼
通知申请人："您的账户已被冻结"（泛化表述，不说明 SAR）
```
