# 移动端 APP 设计方案

## 1. 产品定位

**目标用户**: 美港股投资者（个人投资者为主）
**核心价值**: 便捷交易 + 实时行情 + 资产管理

**竞品参考**
- Robinhood: 极简设计，降低交易门槛
- Tiger Brokers: 专业行情 + 社区
- Futu: 功能全面，信息密度高
- Webull: 图表强大，适合活跃交易者

---

## 2. 核心功能模块

### 2.1 优先级 P0（MVP 必须）

| 模块 | 功能 | 说明 |
|------|------|------|
| **登录/注册** | 手机号登录、邮箱登录、生物识别 | 安全认证 |
| **行情** | 实时报价、K线图、分时图、深度图 | 核心功能 |
| **交易** | 买入/卖出、订单确认、订单列表 | 核心功能 |
| **持仓** | 持仓列表、盈亏统计、资产总览 | 核心功能 |
| **出入金** | 入金、出金、银行卡管理 | 资金流转 |
| **我的** | 个人信息、设置、帮助 | 基础功能 |

### 2.2 优先级 P1（后续迭代）

- 资讯/研报
- 活动页面
- 社区/讨论
- 推荐算法

## 4. 页面设计

### 4.1 登录/注册流程

**页面流程**
```
启动页 → 登录页 → 手机验证 → 设置密码 → 完成
                ↓
              注册页 → KYC 认证 → 完成
```

**登录页设计要点**
- 手机号 + 验证码登录（主要方式）
- 邮箱 + 密码登录（备用方式）
- 生物识别（Face ID / Touch ID）快速登录
- "忘记密码" 和 "注册账号" 入口

## 5. 设计规范

### 5.1 颜色系统

**主色调**
- Primary: `#1890FF` (蓝色，交易/确认按钮)
- Success: `#52C41A` (绿色，涨/买入)
- Danger: `#FF4D4F` (红色，跌/卖出)
- Warning: `#FAAD14` (黄色，警告提示)

**中性色**
- Text Primary: `#262626`
- Text Secondary: `#8C8C8C`
- Border: `#D9D9D9`
- Background: `#F5F5F5`

### 5.2 字体系统

**iOS**
- 标题: SF Pro Display, 24px, Bold
- 正文: SF Pro Text, 16px, Regular
- 辅助: SF Pro Text, 14px, Regular
- 数字: SF Pro Display, 20px, Medium (等宽)

**Android**
- 标题: Roboto, 24sp, Bold
- 正文: Roboto, 16sp, Regular
- 辅助: Roboto, 14sp, Regular
- 数字: Roboto Mono, 20sp, Medium

### 5.3 间距系统

- 4px: 最小间距
- 8px: 组件内间距
- 16px: 组件间间距
- 24px: 区块间间距
- 32px: 页面边距

### 5.4 圆角

- 小圆角: 4px (按钮、输入框)
- 中圆角: 8px (卡片)
- 大圆角: 16px (弹窗)

## 6. 交互规范

### 6.1 手势

- **点击**: 进入详情、选择、确认
- **长按**: 显示更多操作（删除、编辑）
- **左滑**: 快速操作（买入/卖出/删除）
- **下拉**: 刷新数据
- **上拉**: 加载更多

### 6.2 反馈

- **加载**: 显示 Loading 动画
- **成功**: Toast 提示 + 震动反馈
- **失败**: Toast 提示 + 错误说明
- **确认**: 二次确认弹窗（交易、出金）

### 6.3 动画

- 页面切换: 300ms 淡入淡出
- 列表滚动: 流畅滚动，无卡顿
- 数字变化: 数字滚动动画（资产变化）
- 按钮点击: 缩放反馈（0.95x）

## 7. 技术实现

### 7.1 技术栈

**iOS**
- 语言: Swift 5.9+
- UI 框架: SwiftUI + UIKit
- 网络: Alamofire
- WebSocket: Starscream
- 图表: Charts (danielgindi/Charts)
- 存储: CoreData + Keychain

**Android**
- 语言: Kotlin 1.9+
- UI 框架: Jetpack Compose
- 网络: Retrofit + OkHttp
- WebSocket: OkHttp WebSocket
- 图表: MPAndroidChart
- 存储: Room + EncryptedSharedPreferences

### 7.2 架构模式

**MVVM (Model-View-ViewModel)**
```
View (SwiftUI/Compose)
  ↓
ViewModel (业务逻辑)
  ↓
Repository (数据层)
  ↓
API Service / Local DB
```

### 7.3 数据流

**行情数据**
- WebSocket 实时推送（自选股、详情页）
- HTTP 轮询（列表页，5 秒间隔）
- 本地缓存（减少请求）

**交易数据**
- HTTP POST 提交订单
- WebSocket 推送订单状态变化
- 本地持久化订单历史

## 9. 安全设计

### 9.1 认证

- JWT Token 认证
- Token 过期自动刷新
- 生物识别快速登录
- 敏感操作二次验证

### 9.2 数据加密

- HTTPS 传输
- 本地数据加密存储（Keychain/EncryptedSharedPreferences）
- 敏感信息脱敏显示（银行卡号、身份证号）

### 9.3 风控

- 设备指纹识别
- 异常登录检测
- 交易频率限制
- 大额交易二次确认

## 附录：竞品分析

### Robinhood
- **优点**: 极简设计，降低门槛，无手续费
- **缺点**: 功能较少，专业用户不满足
- **借鉴**: 简洁的交易流程，清晰的视觉层级

### Tiger Brokers
- **优点**: 功能全面，社区活跃，多市场支持
- **缺点**: 信息密度高，新手学习成本高
- **借鉴**: 多市场切换，社区功能

### Futu
- **优点**: 行情数据丰富，图表专业，FPS 实时入金
- **缺点**: 界面复杂，操作路径长
- **借鉴**: 专业图表，实时入金体验

### Webull
- **优点**: 图表强大，技术指标丰富，适合活跃交易者
- **缺点**: 学习曲线陡峭
- **借鉴**: 图表交互，技术分析工具
