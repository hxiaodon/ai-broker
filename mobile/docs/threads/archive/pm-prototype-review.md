## 三、阻塞开发的 5 个问题（必须解决）

### 阻塞-1：港股 MVP 范围未最终定稿

**现状**: 技术部建议港股推 Phase 2 可缩短 2-3 个月。但 v3-final 的出入金页面已经设计了双币种（USD/HKD）和换汇功能，行情页有港股 Tab。如果港股推后，这些设计需要全部调整。

**需要决策**:
- 方案 A: Phase 1 包含港股 → 需要补充 Board Lot、竞价时段、HKD 出入金（FPS）、T+2 结算的完整设计
- 方案 B: Phase 1 仅美股 → 需要从原型中移除港股 Tab、HKD 币种、换汇功能，改为"即将上线"

**建议**: **方案 B**。理由：
1. 港股适配增加 30% 工作量（交易引擎评估）
2. FPS 通道完全未设计，HKD 出入金是空白
3. Board Lot 规则在下单页完全未体现，每只港股每手股数不同，需要后端接口支持
4. 先用美股跑通完整闭环，Phase 2 加入港股更稳妥

### 阻塞-2：原型中出金流程缺少生物识别确认

**现状**: 设计文档明确要求出金操作需 Face ID 验证（入金确认弹窗中有"确认并提交 (Face ID)"），但原型的出金流程直接以普通按钮提交，没有生物识别步骤。

**影响**: 这是安全合规硬性要求（`.claude/rules/security-compliance.md` 明确规定 "Required for: fund withdrawal"）。如果原型没有体现，工程师可能按普通按钮实现。

**要求**: 原型必须修正，出金确认弹窗增加 Face ID 验证步骤，与入金保持一致。

### 阻塞-3：游客模式的延迟行情标识完全缺失

**现状**: 设计文档明确要求游客模式下价格旁显示"延迟"徽标，页面顶部提示"行情延迟 15 分钟，登录获取实时行情"。但 market.html 和 stock-detail.html 原型中完全没有体现。

**影响**: SEC 和交易所数据使用协议**禁止将延迟数据展示为实时数据**。这是合规红线。

**要求**: 原型必须增加游客模式状态：
- 顶部通知栏："行情延迟 15 分钟，登录获取实时行情"
- 每个价格旁增加"延迟"小标签

### 阻塞-4：stock-detail.html 缺少买入/卖出按钮

**现状**: 股票详情页是用户从"看行情"到"下单交易"的核心桥梁。设计文档明确在底部有固定的 `[买入] [卖出]` 按钮。但原型完全没有。

**影响**: 这是核心交易链路的断点。用户在详情页看好一只股票后无法直接交易，必须退回到其他页面找入口。

**要求**: 原型必须在 stock-detail.html 底部增加固定的买入/卖出按钮栏。

### 阻塞-5：Admin Panel 需求必须同步启动

**现状**: 技术部统一意见稿 CRITICAL-5 指出 Admin Panel 完全缺失。KYC 人工审核、出金审批、AML 筛查处理、订单监控 — 这些全部需要运营后台支撑，是合规运营的基础设施。

**影响**: 没有 Admin Panel，KYC 审核无法执行（Phase 1 用人工审核替代活体检测，人工审核在哪里做？），出金审批流程无法落地（Rule 5 三级审批），CTR/SAR 申报无工具支撑。

**要求**: 产品侧需在 **1 周内** 输出 Admin Panel 的 P0 模块需求文档：
- KYC 审核工作台（审核队列、证件查看、审批操作）
- 出金审批后台（三级审批流程）
- 用户账户管理
- 基础订单监控

## 五、流程串演验证

逐一走了核心业务流程，以下是结论：

### 流程 1: 新用户注册→开户 ❌ 有断点

```
启动页 → 手机号登录页 → "注册新账户" → ⚠️ 跳过了手机 OTP 验证直接到 KYC
```
**修正**: 注册应先走 OTP 验证 → 自动登录 → 进入 KYC

### 流程 2: KYC 完整流程 ✅ 基本通过

```
个人信息(1/7) → 证件上传(2/7) → 财务状况(3/7) → 投资评估(4/7) → W-8BEN(5/7) → 风险披露(6/7) → 协议签署(7/7) → 审核中
```
7 步流程完整，进度条正确，W-8BEN 税务协定处理到位。**但缺少审核通过/拒绝/补材料三个结果页。**

### 流程 3: 看行情→下单→查订单 ❌ 有断点

```
行情列表 → 点击股票 → 股票详情页 → ⚠️ 无买入/卖出按钮 → 无法进入下单页
```
**修正**: stock-detail.html 必须加底部交易按钮

如果手动跳到 trade.html，后续流程通畅：
```
下单页 → 选择订单类型 → 输入价格/数量 → 滑动确认 → Face ID → 订单成功 → 自动跳转订单列表
```

### 流程 4: 查看持仓→卖出 ✅ 通过

```
持仓页 → 某个持仓卡片 → [卖出] → 跳转 trade.html?action=sell → 显示持仓信息（已结算/未结算）→ 滑动确认
```

### 流程 5: 入金 ✅ 基本通过

```
我的 → 资金管理 → 入金 → 选择方式/银行卡/金额 → 确认弹窗 (含 Face ID) → 提交
```
**但缺少"添加银行卡"流程** — 新用户首次入金无法完成。

### 流程 6: 出金 ❌ 缺少生物识别

```
资金管理 → 出金 → 输入金额 → 确认 → ⚠️ 无 Face ID 验证 → 直接成功
```
**修正**: 必须加 Face ID 确认步骤

### 流程 7: 游客模式 ❌ 无标识

```
启动页 → "先浏览行情" → 行情页 → ⚠️ 没有"延迟15分钟"标识 → 与已登录用户界面无差异
```
