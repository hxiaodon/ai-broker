**已修改文件**: `funding.html`

**设计稿需同步**:

- Face ID 验证中/成功/失败状态设计
- 确保与入金确认弹窗保持视觉一致性

### 3. 游客模式延迟行情标识

**已修改文件**: `market.html`、`stock-detail.html`、`login.html`

**修改内容**:
- `login.html`: "先浏览行情"链接现在带 `?guest=1` 参数
- `market.html`: 游客模式下顶部显示琥珀色通知栏"行情延迟 15 分钟，登录获取实时行情"+ 去登录入口；每个股票价格旁显示"延迟"徽标

**设计稿需同步**:
- 延迟通知栏的完整视觉设计（琥珀色背景 + "延迟"标签 + 文案 + 登录入口）

- 游客模式下的底部交易按钮替换设计（登录引导 Bottom Sheet）
- 确保延迟标识在所有价格展示位置统一

### 4. stock-detail.html 买入/卖出按钮

## 一、需要设计稿同步的变更（4 项）

> **UXUE 完成状态 (2026-03-13)**: 以下 4 项均已在原型代码中完成适配，对应 CHANGELOG v1.6：
> 1. profile.html — 布局确认正确，无需变更 ✅
> 2. settings.html — 退出登录 Bottom Sheet 已替换系统 confirm() ✅
> 3. portfolio.html — 持仓股票 hover/active 交互反馈已添加 ✅
> 4. market.html — 游客 Tab 拦截 Bottom Sheet 已替换系统 confirm() ✅

### 1. "我的"页面精简（profile.html — v1.4）

**变更内容**:
- 移除蓝色渐变资产卡片（总资产/可用/持仓/今日盈亏）及入金/出金快捷按钮 — 避免与"持仓"页重复
- 银行卡管理、出入金记录合并为一个"**资金管理**"入口（副文案"入金 · 出金 · 银行卡"），指向 funding.html 主页
- 安全设置、通用设置合并为一个"**设置**"入口（副文案"通用 · 安全 · 交易"），指向 settings.html

**设计变更理由**: "我的"页定位为纯账户管理入口（对标 Robinhood），资产展示统一收归"持仓"页，菜单项避免包含关系冗余。

**设计稿需同步**:
- 更新"我的"页面 Figma 设计稿，移除资产卡片
- 菜单结构改为：资金管理 / 交易确认书(即将上线) / 消息通知(即将上线) / 设置 / 帮助中心 / 联系客服 / 关于我们
- 副文案样式规范：`text-xs text-gray-400`，跟在菜单标题右侧

### 2. 设置页退出登录按钮（settings.html — v1.5）

**变更内容**:
- 在交易设置分组下方、版本号上方新增"退出登录"按钮
- 确认弹窗 → 跳转 login.html

**设计稿需同步**:
- 退出登录按钮样式：白底圆角卡片 (`rounded-xl`)，红色文字 (`text-red-500`)，居中，全宽
- 点击后 confirm 确认弹窗设计

### 3. 持仓股票可点击（portfolio.html — v1.5）

**变更内容**:
- 持仓股票信息区域（股票名称、价格、盈亏）从不可点击 `<div>` 改为可点击 `<a>` 链接
- 点击跳转到 `stock-detail.html?symbol=XXX` 查看 K 线/新闻/财报

**设计稿需同步**:
- 标注持仓股票卡片的点击热区（整个信息区域，不含底部买入/卖出按钮）
- 添加 hover/active 状态反馈（可沿用行情列表的交互样式）

### 4. 游客模式底部 Tab 拦截弹窗（market.html — v1.5）

**变更内容**:
- 游客模式下点击底部"订单""持仓""我的" Tab 不再直接跳转
- 弹出系统级确认弹窗："登录后即可使用该功能，现在去登录？"
- 确认 → login.html，取消 → 留在行情页

**设计稿需同步**:
- 游客模式下底部 Tab 的视觉状态（保持灰色不可用外观，或加锁图标）
- 拦截弹窗设计（可设计为自定义 Bottom Sheet 替代系统 confirm，体验更一致）
